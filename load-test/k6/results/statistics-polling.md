# 시나리오 3: 통계 조회 폴링 (statistics-polling)

## 목적
가장 빈번하게 호출되는 통계 조회 API의 읽기 성능과 캐시 효과 측정.
50개 방에 분산된 캐시 키 환경에서 현실적인 부하 확인.

## 테스트 설계

**setup**: 방 50개 생성, 각 방에 참여자 10명 등록 + 투표 완료 (총 500명, 7일 x 48슬롯 = 336개 datetime)
**본 테스트**: 50개 방 중 랜덤 선택하여 통계 조회 반복 (sleep 0.5s)

- Caffeine 캐시 TTL 500ms → 초당 캐시 키당 최대 2회 미스
- 50개 캐시 키 → 초당 최대 100회 캐시 미스 (DB 쿼리 발생)
- VU당 1 iteration = API 호출 1회 (통계 조회)
- VU당 1 iteration 소요 시간 ~ 0.55초

## 1~2차: 150 VU (안정성 확인)

| 구간 | VU |
|------|-----|
| Ramp-up | 0->20, 30s |
| Push | 20->50, 1m |
| Stress | 50->100, 1m |
| Max | 100->150, 1m |
| Cool-down | 150->0, 30s |

| 지표 | 1차 | 2차 |
|------|-----|-----|
| 총 요청 | 29,775 | 29,455 |
| RPS | 111/s | **110/s** |
| p50 | 44ms | **43ms** |
| p95 | 224ms | **274ms** |
| p99 | 419ms | **580ms** |
| max | 1.5s | 1.54s |
| 에러율 | **0%** | **0%** |
| 통계 조회 | 28,725회 | 28,405회 |
| 소요 시간 | 4분 28초 | 4분 28초 |

### 분석
- 150 VU에서도 **에러율 0%, p50 = 43~44ms**로 안정적
- 읽기 전용이므로 쓰기(시나리오 1, 2) 대비 월등히 높은 VU 수용 가능
- p99 = 419~580ms 범위 → 캐시 미스 + GC STW 복합 영향
- Caffeine 캐시가 효과적으로 작동하여 DB 부하를 흡수
- 2회 실행 결과 일관성 확인 (RPS, p50 거의 동일)
- **결론: 150 VU까지 안정. 3차에서 더 높은 VU로 한계 탐색**
