# 사전 탐색: 통계 조회 (가벼운 데이터)

방 1개, 참여자 10명, 슬롯 16개(2일 x 8시간)의 가벼운 데이터로 캐시 동작과 기본 응답 특성을 파악하기 위해 실행한 사전 탐색 테스트입니다.

## 조건
- **setup**: 방 1개, 참여자 10명, 2일 x 8슬롯 = 16개 datetime 조합
- **본 테스트**: 같은 방 1개에 대해 통계 조회 반복 (sleep 0.5s)
- 캐시 키 1개 -> 322 req/s에서도 캐시 미스 초당 ~2회 -> **사실상 캐시 성능 테스트**

## 결과 요약

| 지표 | 40 VU | 100 VU | 300 VU |
|------|-------|--------|--------|
| 총 요청 | 8,775 | 37,073 | 96,811 |
| RPS | 32/s | 103/s | **322/s** |
| p50 | 18.8ms | 15.4ms | **15.4ms** |
| p95 | 34.6ms | 42.4ms | **53.8ms** |
| p99 | 1.04s | 1.06s | **1.07s** |
| max | 1.23s | 2.7s | 3.32s |
| 에러율 | 0% | 0% | **0%** |

## 분석
- p50/p95는 VU 수(40->300)에 거의 무관하게 안정적 -> **캐시 히트 시 응답이 일정**
- p99가 40~300 VU 모두 ~1.06s로 동일 -> **GC STW pause로 추정** (VU 수에 비례하지 않음)
- Tomcat 15스레드 + Hikari 5커넥션으로 300 VU, 322 req/s를 에러 0%로 처리
- 데이터가 너무 가벼워 **DB 부하가 거의 없는 상태**이므로, 현실적 조건에서 재테스트 필요
